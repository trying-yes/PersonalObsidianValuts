并不是只有 Cloudflare 有这项技术，但在自建服务器（Home Lab）圈子里，它确实是**出镜率最高、最推荐**的选择。

为了不再觉得“混乱”，可以把这类技术统称为**“内网穿透”**（Intranet Penetration）或**“隧道技术”**（Tunneling）。

---

### 1. 除了 Cloudflare，还有哪些选择？

市面上有很多同类产品，根据原理和使用场景可以分为三类：

- **老牌穿透工具：** `frp` (国产神作)、`ngrok`。
    
    - **特点：** 你需要自己准备一台带有“公网 IP”的云服务器（VPS）作为中继站。
        
    - **缺点：** 增加了购买 VPS 的成本，且需要一定的 Linux 配置基础。
        
- **组网/VPN 类：** `Tailscale`、`ZeroTier`。
    
    - **特点：** 创建一个虚拟的“大局域网”。
        
    - **适用场景：** 主要是为了让你**自己**在外面能连回家的电脑（比如 SSH、看家里的照片），而不是为了把网站公开给**所有人**看。
        
- **SAAS 隧道类：** `Cloudflare Tunnel`。
    
    - **特点：** 厂商提供中继服务器，你只需要安装客户端。
        

---

### 2. 为什么大家都选 Cloudflare？（核心优势）

Cloudflare 之所以能成为“大众情人”，主要是因为它把一堆复杂且昂贵的功能**打包免费**送给你了：

|**优势**|**为什么这很重要**|
|---|---|
|**完全免费**|即使你没有公网 IP，也不想买 VPS，只要有个域名，CF Tunnel 就能白嫖。|
|**自带 SSL (HTTPS)**|别家工具可能需要你自己去折腾证书，CF 自动帮你搞定，地址栏那个“小锁头”直接到位。|
|**极致的安全防护**|**隐藏真实 IP**。别人攻击你的域名，流量会被 CF 挡住。如果是你自己映射端口，黑客可能直接打爆你家路由器。|
|**零配置防火墙**|你可以在网页端设置：“只有中国 IP 能访问”或者“访问时必须输入谷歌账号验证”。|
|**无需配置路由器**|传统的 DDNS 需要去路由器开“端口转发”，CF Tunnel 不需要，只要服务器能上网就行。|

---

### 3. Cloudflare 的“代价”是什么？

虽然它很强，但也有人因为以下原因选择其他方案：

- **延迟问题：** 因为流量要绕道 Cloudflare 的节点（通常在海外或香港），如果你在大陆访问，速度可能不如直连或国内 VPS 中转快。
    
- **隐私担忧：** 你所有的流量都会经过 Cloudflare 检查（它能看到你的流量）。如果你非常在意“数据完全掌握在自己手里”，可能会选 `frp`。
    
- **合规限制：** 某些内容（如纯视频流媒体）在免费版 Tunnel 上可能有流量限制或服务条款约束。
    

---

### 4. 给你的建议（如何避坑）

如果你现在的目标是**“把自建的服务快速、安全地发布到外网”**，Cloudflare 依然是你的最佳起点。

> **底层逻辑梳理：**
> 
> 1. **域名** = 地址（你已经买了）。
>     
> 2. **DNS** = 地图册（你把域名交给 Cloudflare 管理）。
>     
> 3. **Cloudflare Tunnel** = 一条从你家服务器通往 Cloudflare 的“秘密地道”。
>     
> 4. **公众访问** = 用户访问域名 -> 进到 CF 节点 -> 钻入地道 -> 到达你家服务器。
>
